## Objetivo:
Debes diseñar la lógica para que el Agente de IA genere un "Plan de Ejecución Blindado" que el script de Python pueda ejecutar de forma autónoma mientras la IA "duerme".

## Instrucciones Detalladas para el Agente:

1. Gestión de Estrategia por "Slots":
* Operar con un solo bloque de capital. Actualmente el segundo bot opero con 500 USDT que invirtió en SOL y aunque el usuario lo detuvo y ejecuto nuevamente pasandole otro bloque de capital de 100 USDT este no lo invirtió hasta que recupere la primer inversión, ese pensamiento esta muy bien planteado, mantenlo.
* Cada operación abierta debe quedar "sellada" con la estrategia inicial (ej. Swing) que el primer bot le recomendo. Si el Bot 1 cambia la tendencia general después, la operación abierta mantiene su plan original hasta su cierre o expiración por tiempo (ej. 2 meses).
* Dicho lo anterior el segundo bot debe tener la capacidad de ver la estrategia recomendada por el primer bot y realizar un plan de ejecución y determinar el tiempo de ejecución 2 meses y asi, usando el capital proporcionado por el usuario, y hacer uso inteligente de ese capital para generar en el menor tiempo posible la mayor rentabilidad posible.

2. Generación de Órdenes Condicionales (El Plan):
Cada vez que el Agente razone, NO debe dar una opinión, debe devolver un objeto JSON con:

* Punto de Entrada Ideal (Buy Limit): Si el precio actual es mayor, el script de Python espera a que baje a este punto para comprar sin preguntar.
* Take Profit Dinámico (TP): Precio objetivo para vender.
* Stop Loss de Emergencia (SL): Basado en el "colchón" (min/max) y la volatilidad.
* Trailing Stop: Un porcentaje de retroceso desde el máximo alcanzado para proteger ganancias si el precio toca el TP y sigue subiendo.

3. Lógica del Script de Python (El Músculo - 1 min):
Define la lógica para que el script de Python monitoree cada minuto:

* Estado: Esperando Compra. Si precio_actual <= Buy_Limit, ejecutar compra.
* Estado: En Posición. Si precio_actual >= TP o precio_actual <= SL, ejecutar venta inmediata.
* Trigger de Emergencia: Si el precio sale del "colchón" antes del timeframe, despertar al Agente de IA para una re-evaluación forzada.

4. Uso de Timeframes:

* El Agente solo se ejecuta al cierre de cada vela según el timeframe (1h, 4h, 1d) recomendado por el Bot 1.
* Durante la ejecución, el Agente debe validar el balance histórico y actual para asegurar que no se exceda el "Capital Inteligente".

5. Variables Críticas a Incluir:

* Expiración de Operación: Tiempo máximo para mantener la posición.
* Ratio Riesgo/Beneficio: Validar que la estrategia del Bot 1 sea lógica antes de entrar.

Proporcióname el código en Python que implemente esta separación de niveles (Cerebro vs. Músculo) y el formato de respuesta (JSON) que el Agente de IA debe seguir para que el script nunca pierda una oportunidad de compra o venta por esperar al razonamiento.
